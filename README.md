# 📖 Sobre o Projeto

Este projeto teve como objetivo utilizar o **Google NotebookLM** como uma ferramenta de apoio aos estudos para a certificação **Microsoft SC-300 – Identity and Access Administrator**.

Foram reunidas diferentes fontes de conhecimento, como documentação oficial da Microsoft e vídeos do YouTube, permitindo ao NotebookLM centralizar as informações em um único ambiente para gerar resumos, responder perguntas e apoiar revisões.

Além de estudar para a certificação, o projeto buscou demonstrar como ferramentas de Inteligência Artificial podem acelerar o aprendizado quando combinadas com fontes confiáveis e boas práticas de engenharia de prompts.

---

# 🎯 Objetivos

- Explorar o NotebookLM como ferramenta de aprendizagem.
- Consolidar conhecimentos para a certificação SC-300.
- Aprender técnicas de engenharia de prompts.
- Produzir um material de revisão reutilizável.
- Organizar informações provenientes de diferentes fontes.

# ⚠️ Troubleshooting (Cicatrizes)

Durante o uso do NotebookLM foram identificadas algumas dificuldades e os respectivos ajustes realizados.

| Situação | Solução |
|----------|----------|
| Respostas muito genéricas | Adicionar contexto da certificação SC-300 |
| Poucos exemplos | Solicitar cenários corporativos |
| Resumos superficiais | Solicitar estrutura em tópicos |
| Respostas extensas | Solicitar respostas objetivas |
| Conceitos semelhantes | Pedir comparações em formato de tabela |

Esses ajustes melhoraram significativamente a qualidade das respostas geradas pela IA.

# 📘 Miniguia de Estudos

## Microsoft Entra ID

Serviço de gerenciamento de identidade e acesso da Microsoft.

### Principais funcionalidades

- Single Sign-On (SSO)
- Multi-Factor Authentication (MFA)
- Conditional Access
- Identity Protection
- Identity Governance
- Privileged Identity Management

---

## Authentication

Validação da identidade do usuário utilizando senha, MFA, biometria ou certificados.

---

## Authorization

Processo responsável por determinar quais recursos poderão ser acessados pelo usuário autenticado.

---

## Conditional Access

Permite criar políticas baseadas em:

- Usuário
- Grupo
- Aplicação
- Localização
- Dispositivo
- Risco

---

## Privileged Identity Management

Permite acesso administrativo temporário (Just-In-Time), reduzindo privilégios permanentes.

---

## Identity Governance

Responsável pelo gerenciamento do ciclo de vida das identidades e permissões.

---

# 📚 Glossário

| Termo | Definição |
|--------|-----------|
| Entra ID | Plataforma de identidade da Microsoft |
| MFA | Multi-Factor Authentication |
| SSO | Single Sign-On |
| RBAC | Role-Based Access Control |
| PIM | Privileged Identity Management |
| Conditional Access | Políticas de acesso baseadas em condições |
| Identity Governance | Governança do ciclo de vida das identidades |
